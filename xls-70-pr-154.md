**Comment:** Remove the ignore — the integration test is not running in CI.

**Fix:** This test had a hardcoded fictitious credential hash that would fail as soon as the ignore was dropped. Rewrote the test to:
1. Provision issuer + subject wallets
2. Submit `CredentialCreate` (issuer → subject) + `CredentialAccept` (subject)
3. Set up `DepositPreauth` on the destination authorizing the credential
4. Fetch the real credential hash from `account_objects`
5. Pass the real hash to `deposit_authorized`
6. Assert the response echoes the credential

`#[ignore]` removed.

**Draft reply:**
```
The test body needed a rewrite first — it was using a hardcoded fictitious credential ID
("documents the expected shape"). Simply removing #[ignore] would fail because rippled
rejects the call when the credential doesn't exist. Rewrote to provision a real
CredentialCreate + CredentialAccept and read the hash from account_objects before querying.
#[ignore] dropped and test passes.
```
